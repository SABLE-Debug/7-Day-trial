# Pass-Over Prompt

Copy the block below into a **new chat inside your MOBIUS Claude.ai Project**.
Attach any assets (logos, copy docs, screenshots, reference sites) to the same
message. This is the prompt that picks up where the Claude Code session left off.

---

I'm continuing the MOBIUS brand studio project. Context is in this Project's
knowledge:

- `DESIGN-SYSTEM.md` — the visual language: colors, type, spacing, motion, copy
  tone. Follow it exactly.
- `reference-site.html` — the landing page already shipped. This is the quality
  bar and the source of truth for the MOBIUS look. Open it to see the patterns.
- `PROJECT-INSTRUCTIONS.md` — how you should work and what to output.

What's done: an 18-section, scroll-animated MOBIUS landing page is live.

What I want now: [DESCRIBE THE TASK — e.g. "a pricing page", "rework the hero",
"a pitch deck", "an email template"]. I've attached: [LIST YOUR ASSETS, or write
"nothing yet — ask me for what you need"].

Before writing code:
1. Tell me the section structure and motion beats you'd build.
2. Flag anything in my brief or assets that fights the MOBIUS design system.
3. Suggest improvements I haven't asked for — copy, layout, sections.

Then deliver it as a **single self-contained HTML file in an Artifact** so I can
preview it live in this chat. Keep iterating in that same Artifact.

---

## Tips for using this in normal chat

- The Artifact panel renders HTML live — scroll, click, resize it right there.
  That solves the "I can't preview" problem from the Claude Code session.
- Drag assets straight onto the message. Screenshots of sites you like work
  great as references — Claude can match a vibe from an image.
- To download: the Artifact has a copy/download button; save as `.html` and
  open it in any browser, or drop it back into the GitHub repo.
- Start a fresh chat per major surface (one for the pricing page, one for the
  deck). The Project knowledge keeps them all consistent automatically.
